# Команды микроконтроллера ATmega328 (сателлит)

## Протокол

Пакет: `[ADDR][LEN][CMD][DATA...][CRC16_LO][CRC16_HI]`
- ADDR: 1–32 (адрес сателлита по DIP-переключателям)
- LEN: количество байт от CMD до конца DATA включительно (не считая CRC)
- CRC16: Modbus CRC считается по всем байтам от ADDR до последнего байта DATA
- Все ответы имеют ту же структуру

**Как приёмник определяет конец пакета:**
1. Прочитать байт ADDR
2. Прочитать байт LEN
3. Прочитать ровно LEN байт (CMD + DATA)
4. Прочитать 2 байта CRC16
5. Итого всегда: `LEN + 4` байт — граница пакета известна точно

---

## Таблица команд

LEN запроса = 1 (только CMD, без DATA) если DATA пустой, иначе 1 + len(DATA).

| CMD  | Название         | Направление     | LEN req | LEN resp | DATA запроса | DATA ответа              |
|------|------------------|-----------------|---------|----------|--------------|--------------------------|
| 0x01 | PING             | PC → MCU → PC   | 1       | 2        | —            | `[0x01]` (ACK)           |
| 0x02 | GET_STATUS       | PC → MCU → PC   | 1       | 2        | —            | `[FLAGS]`                |
| 0x03 | MOTOR_START      | PC → MCU → PC   | 2       | 2        | `[DIR]`      | `[0x01]` (ACK)           |
| 0x04 | MOTOR_STOP       | PC → MCU → PC   | 1       | 2        | —            | `[0x01]` (ACK)           |
| 0x05 | DISPENSE_RESULT  | PC → MCU → PC   | 1       | 2        | —            | `[RESULT]`               |
| 0x06 | GET_ADDR         | PC → MCU → PC   | 1       | 2        | —            | `[ADDR]`                 |
| 0x07 | BUTTON_EVENT     | MCU → PC (push) | —       | 2        | —            | `[0x07][ADDR]`           |
| 0x08 | SET_SENSOR_MODE  | PC → MCU → PC   | 2       | 2        | `[MODE]`     | `[0x01]` (ACK)           |
| 0xFF | ERROR            | MCU → PC        | —       | 2        | —            | `[ERR_CODE]`             |

---

## Описание команд и алгоритмы

---

### 0x01 — PING

**Назначение:** проверка связи с сателлитом.

**Алгоритм MCU:**
1. Принять пакет, проверить CRC16.
2. Если CRC не совпал → отправить `[ADDR][0xFF][0x01][CRC]` (ERR: bad CRC).
3. Если адрес совпадает с DIP → отправить `[ADDR][0x01][0x01][CRC]` (ACK).

---

### 0x02 — GET_STATUS

**Назначение:** получить текущее состояние сателлита.

**Байт FLAGS:**
| Бит | Значение |
|-----|----------|
| 0   | Мотор активен (1 = крутится) |
| 1   | Детектор выдачи сработал (1 = сработал) |
| 2   | Кнопка нажата (1 = нажата) |
| 3–7 | Зарезервировано |

**Алгоритм MCU:**
1. Принять пакет, проверить CRC16.
2. Собрать байт FLAGS из текущих состояний пинов PD4 (мотор), PD3 (детектор), PB5 (кнопка).
3. Отправить `[ADDR][0x02][FLAGS][CRC]`.

---

### 0x03 — MOTOR_START

**Назначение:** запустить мотор для выдачи товара.

**DATA:** `[DIR]` — направление: `0x00` = прямое (PD5=LOW), `0x01` = реверс (PD5=HIGH).

**Алгоритм MCU:**
1. Принять пакет, проверить CRC16.
2. Установить PD5 согласно DIR.
3. Установить PD4=HIGH (мотор включён).
4. Запустить таймер выдачи (по умолчанию 5 сек — настраивается).
5. Взвести флаг ожидания детектора.
6. Отправить `[ADDR][0x03][0x01][CRC]` (ACK).
7. В основном цикле: если PD3 сработал → сохранить флаг DISPENSE_OK.
8. Если таймер истёк без срабатывания PD3 → сохранить флаг DISPENSE_TIMEOUT, остановить мотор (PD4=LOW).

---

### 0x04 — MOTOR_STOP

**Назначение:** принудительно остановить мотор.

**Алгоритм MCU:**
1. Принять пакет, проверить CRC16.
2. Установить PD4=LOW (мотор выключен).
3. Сбросить таймер выдачи и флаг ожидания детектора.
4. Отправить `[ADDR][0x04][0x01][CRC]` (ACK).

---

### 0x05 — DISPENSE_RESULT

**Назначение:** запросить результат последней попытки выдачи.

**Байт RESULT:**
| Значение | Описание |
|----------|----------|
| 0x00     | Нет данных (выдача не запускалась) |
| 0x01     | DISPENSE_OK — детектор сработал |
| 0x02     | DISPENSE_TIMEOUT — товар не вышел |
| 0x03     | Мотор ещё работает |

**Алгоритм MCU:**
1. Принять пакет, проверить CRC16.
2. Прочитать сохранённый флаг результата выдачи.
3. Отправить `[ADDR][0x05][RESULT][CRC]`.
4. Сбросить флаг результата (готов к следующей выдаче).

---

### 0x06 — GET_ADDR

**Назначение:** прочитать адрес сателлита (для диагностики DIP-переключателей).

**Алгоритм MCU:**
1. Принять пакет, проверить CRC16.
2. Прочитать биты PB0–PB4 (INPUT_PULLUP, активный уровень LOW).
3. Вычислить адрес: `addr = (~PINB & 0x1F)`.
4. Отправить `[ADDR][0x06][addr][CRC]`.

---

### 0x07 — BUTTON_EVENT (unsolicited response)

**Назначение:** сателлит самостоятельно сообщает о нажатии кнопки вызова.

> Это единственная команда, инициируемая MCU без запроса от PC.
> MCU отправляет пакет только когда шина свободна (после завершения текущего цикла опроса).
> Реализуется через флаг: в прерывании INT на PB5 взводится флаг `btn_pending`, в основном цикле — отправка.

**Алгоритм MCU:**
1. Прерывание по фронту на PB5 (кнопка нажата, INPUT_PULLUP → LOW).
2. Антидребезг: подождать 20 мс, проверить PB5 снова.
3. Если PB5 всё ещё LOW → взвести флаг `btn_pending`.
4. В основном цикле: если `btn_pending` и шина свободна (DE=LOW) → отправить `[ADDR][0x07][ADDR][CRC]`.
5. Сбросить `btn_pending`.

---

### 0x08 — SET_SENSOR_MODE

**Назначение:** настроить режим работы датчика выдачи (прямой/реверс).

**DATA:** `[MODE]` — `0x00` = нормально-открытый (срабатывание по LOW), `0x01` = нормально-закрытый (срабатывание по HIGH).

**Алгоритм MCU:**
1. Принять пакет, проверить CRC16.
2. Сохранить MODE в EEPROM (адрес 0x00).
3. Обновить переменную `sensor_mode` в RAM.
4. Отправить `[ADDR][0x08][0x01][CRC]` (ACK).

---

### 0xFF — ERROR

**Назначение:** MCU сообщает об ошибке в ответ на некорректный запрос.

**Байт ERR_CODE:**
| Значение | Описание |
|----------|----------|
| 0x01     | Ошибка CRC |
| 0x02     | Неизвестная команда |
| 0x03     | Команда недопустима в текущем состоянии (мотор уже работает) |

**Алгоритм MCU:**
1. При обнаружении ошибки (плохой CRC, неизвестный CMD, конфликт состояния) → отправить `[ADDR][0xFF][ERR_CODE][CRC]`.

---

## Основной цикл MCU (псевдокод)

```
loop:
    если есть байты в UART RX буфере:
        прочитать байт → ADDR
        прочитать байт → LEN
        прочитать LEN байт → buf[]   // CMD + DATA
        прочитать 2 байта → crc_recv
        вычислить CRC16(ADDR, LEN, buf[])
        если crc_recv != crc_calc → отправить ERROR 0x01, continue
        если ADDR != my_addr → игнорировать, continue
        выполнить обработчик buf[0] (CMD)

    если btn_pending и DE == LOW:
        отправить BUTTON_EVENT
        сбросить btn_pending

    если мотор активен:
        если детектор сработал (с учётом sensor_mode):
            PD4 = LOW (стоп)
            dispense_result = DISPENSE_OK
        если таймер истёк:
            PD4 = LOW (стоп)
            dispense_result = DISPENSE_TIMEOUT
```
