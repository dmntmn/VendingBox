# Центральное ПО управляющего компьютера

## Стек

- **Язык:** Python 3.11+
- **Асинхронность:** asyncio + asyncio.Queue
- **COM-порты:** pyserial-asyncio
- **UI:** PyQt6 + QMediaPlayer
- **БД:** SQLite (встроенный sqlite3)
- **Конфигурация:** JSON

## Типы событий в очереди

Все драйверы кладут в `Q_IN` унифицированные события одного формата:

```python
{
    "source": "rs485" | "ccnet" | "mdb",
    "event":  "BUTTON_PRESSED" | "BILL_ESCROW" | ...,
    "data":   { ... }  # зависит от события
}
```

| source | event | data | Описание |
|--------|-------|------|----------|
| `rs485` | `BUTTON_PRESSED` | `{"addr": 5}` | Нажата кнопка на сателлите №5 |
| `rs485` | `DISPENSE_OK` | `{"addr": 5}` | Детектор выдачи сработал |
| `rs485` | `DISPENSE_TIMEOUT` | `{"addr": 5}` | Мотор крутился, товар не вышел |
| `ccnet` | `BILL_ESCROW` | `{"amount": 100}` | Купюра на удержании |
| `ccnet` | `BILL_STACKED` | `{"amount": 100}` | Купюра принята |
| `ccnet` | `BILL_RETURNED` | `{}` | Купюра отклонена |
| `mdb` | `SESSION_BEGIN` | `{}` | Карта приложена |
| `mdb` | `VEND_APPROVED` | `{"amount": 150}` | Оплата картой одобрена |
| `mdb` | `VEND_DENIED` | `{}` | Оплата отклонена |

## Конечный автомат (FSM)

```mermaid
stateDiagram-v2
    [*] --> IDLE : старт приложения

    IDLE --> ITEM_SELECTED : rs485 BUTTON_PRESSED
    note right of IDLE : экран — реклама\nвсе платёжные устройства отключены

    ITEM_SELECTED --> WAITING_PAYMENT : активированы CC-Net и MDB
    note right of ITEM_SELECTED : экран — цена товара\nтаймер 30 сек

    WAITING_PAYMENT --> IDLE : таймаут 30 сек
    WAITING_PAYMENT --> ITEM_SELECTED : rs485 BUTTON_PRESSED (смена товара)
    WAITING_PAYMENT --> DISPENSING : оплата подтверждена

    DISPENSING --> SUCCESS : rs485 DISPENSE_OK
    DISPENSING --> ERROR : rs485 DISPENSE_TIMEOUT

    SUCCESS --> IDLE : экран — «Заберите товар» 5 сек
    ERROR --> REFUND : вернуть деньги
    REFUND --> IDLE : возврат выполнен
```

## Взаимодействие корутин через очередь

```mermaid
sequenceDiagram
    participant RS as driver_rs485
    participant Q as asyncio.Queue
    participant FSM as fsm
    participant CC as driver_ccnet
    participant MDB as driver_mdb
    participant UI as ui (PyQt6)

    RS->>Q: {source:rs485, event:BUTTON_PRESSED, data:{addr:5}}
    Q->>FSM: получить событие
    FSM->>UI: Q_UI → показать цену товара №5
    FSM->>CC: включить приём купюр (ENABLE)
    FSM->>MDB: начать сессию (SETUP + POLL)

    alt Оплата наличными
        CC->>Q: {source:ccnet, event:BILL_ESCROW, data:{amount:100}}
        Q->>FSM: получить событие
        FSM->>CC: STACK (принять купюру)
        CC->>Q: {source:ccnet, event:BILL_STACKED, data:{amount:100}}
        Q->>FSM: сумма достаточна → переход DISPENSING
    else Оплата картой
        MDB->>Q: {source:mdb, event:SESSION_BEGIN}
        Q->>FSM: получить событие
        FSM->>MDB: VEND REQUEST [сумма]
        MDB->>Q: {source:mdb, event:VEND_APPROVED}
        Q->>FSM: переход DISPENSING
    end

    FSM->>RS: команда MOTOR_START [addr=5]
    FSM->>UI: Q_UI → «Выдача товара...»

    RS->>Q: {source:rs485, event:DISPENSE_OK, data:{addr:5}}
    Q->>FSM: переход SUCCESS
    FSM->>DB: записать транзакцию
    FSM->>MDB: VEND SUCCESS / SESSION COMPLETE
    FSM->>CC: отключить приём купюр
    FSM->>UI: Q_UI → «Заберите товар»
```

## Драйвер RS-485 — логика опроса

Драйвер работает в бесконечном цикле, опрашивая все 32 адреса. При обнаружении события кладёт его в очередь.

```mermaid
flowchart TD
    START([asyncio корутина старт]) --> INIT[открыть COM-порт\npyserial-asyncio]
    INIT --> LOOP[addr = 1]
    LOOP --> SEND[await отправить запрос addr]
    SEND --> RECV[await ответ с таймаутом 10мс]
    RECV --> CHK{ответ получен?}
    CHK -->|нет| NEXT[addr++]
    CHK -->|да| PARSE[разобрать пакет Modbus CRC]
    PARSE --> EVT{есть событие?}
    EVT -->|нет| NEXT
    EVT -->|да| PUT[await Q_IN.put - событие -]
    PUT --> NEXT
    NEXT --> END{addr > 32?}
    END -->|нет| LOOP
    END -->|да| LOOP
```

## Запуск asyncio и PyQt6 совместно

PyQt6 имеет собственный event loop, asyncio — свой. Для совместной работы используется `qasync`:

```python
# main.py
import asyncio
import qasync
from PyQt6.QtWidgets import QApplication
from ui.ui import MainWindow
from fsm import FSM

async def main():
    fsm = FSM()
    await asyncio.gather(
        fsm.run(),               # центральный автомат
        fsm.rs485.run(),         # корутина RS-485
        fsm.ccnet.run(),         # корутина CC-Net
        fsm.mdb.run(),           # корутина MDB
    )

app = QApplication([])
loop = qasync.QEventLoop(app)
asyncio.set_event_loop(loop)
with loop:
    loop.run_until_complete(main())
```
